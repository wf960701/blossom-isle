# PRD 模板 - 产品需求文档

> 使用此模板输出标准 PRD，每个决策点需给出 2-3 个选项。

---

## 1. 产品概述

```
产品名称：中文名 / English Name
产品定位：一句话说明这是什么产品
目标用户：谁会用这个产品
核心价值：解决什么痛点
```

## 2. 用户故事

```
作为 [角色]，我想要 [功能]，以便 [价值]。

示例：
作为读者，我想要按标签筛选文章，以便快速找到感兴趣的内容。
作为管理员，我想要在后台写文章，以便发布到网站上。
```

## 3. 功能清单

### P0 - 核心功能（必须做）
- [ ] 功能 1：描述
- [ ] 功能 2：描述

### P1 - 重要功能（尽量做）
- [ ] 功能 3：描述

### P2 - 锦上添花（未来再做）
- [ ] 功能 4：描述

## 4. 页面路由

| 路由 | 页面说明 | 类型 | 访问权限 |
|------|---------|:----:|:--------:|
| `/` | 首页 | 公开 | 所有人 |
| `/login` | 登录页 | 公开 | 所有人 |

## 5. 数据模型

```sql
-- 核心表
table_name (
  id       BIGSERIAL PRIMARY KEY,
  field    TYPE NOT NULL,
  ...
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 6. 非功能需求

- 响应式：移动端 + 桌面端
- 性能：页面加载 < 2s
- 安全：仅管理员可写
- SEO：页面支持 meta 标签
